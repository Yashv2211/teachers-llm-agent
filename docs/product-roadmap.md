## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** The first phase sets up everything so feature work in later phases is fast and clean.
3. **Magic moment first.** A teacher creates an agent in 10 minutes, a student scans a QR code, gets a Swahili answer. This is the milestone everything builds toward.
4. **Test as you go.** Don't save testing for the end. Each task includes verification steps.
5. **Progressive enhancement.** Start with the simplest working version, then layer on polish.
6. **Review before proceeding.** Push each completed phase as a PR and let a review agent (e.g. [CodeRabbit](https://coderabbit.ai) — free for open source) review it before starting the next phase.
7. **Free tier discipline.** Every integration decision must stay within free tiers. Groq Whisper is the only cost risk at scale — flag usage in grant applications before it becomes a problem.

## Agent Session Guide

### How to Use This Roadmap with Claude Code

1. **Start a session:** Give Claude Code the phase prompt at the beginning of each phase. You'll find it at the top of each phase section.
2. **Read selectively:** Each phase lists its Reference sections — the specific parts of `docs/prd.md` and `docs/product-vision.md` Claude needs for that phase. Only read those sections, not the full documents.
3. **Let it work:** Claude reads the roadmap, finds the first unchecked task, implements it, and marks it `[x]` in the roadmap file.
4. **One session = one phase:** Try to complete a full phase in one session. If you stop mid-phase, resume with: "Read `docs/product-roadmap.md`. Find where we left off and continue."
5. **Push a PR for review:** When a phase is complete, push as a PR and let [CodeRabbit](https://coderabbit.ai) review it (free for open source) before starting the next phase.
6. **Update the status line:** After completing tasks, update the header: `**Status:** X/52 tasks complete`.

### Prompt Templates

**Starting a new phase:**
> "Read `docs/product-roadmap.md` and find the current phase. Read only the Reference sections listed for that phase from `docs/prd.md` and `docs/product-vision.md`. Start working on the first unchecked task. After completing each task, update the checkbox to `[x]` in the roadmap. Continue through the phase."

**Resuming after a break:**
> "Read `docs/product-roadmap.md`. Find the first unchecked task. Read only the Reference sections listed for the current phase. Continue from there."

**After completing a phase:**
> "Phase [N] is complete. Create a branch `phase-[N]/[slug]`, commit all changes, push, and open a PR targeting main with the phase goal and completed task count in the body."

**Fixing an issue:**
> "There's a problem with [description]. Read the relevant section of `docs/prd.md` for the expected behavior and fix it. Don't mark any new tasks complete until the fix is verified."

### Phase Review Workflow

After every phase, push a PR before starting the next one. Use this PR format:

```
Title: Phase [N]: [Phase Title]

## Goal
[The phase goal statement]

## Completed
[X] tasks (TASK-[first] through TASK-[last])

## What to verify
- [Key things to test manually]

```

[CodeRabbit](https://coderabbit.ai) is free for open source repos and provides automated review on every PR. Install it on the GitHub repo before Phase 1. It will catch type issues, security problems, and patterns the coding agent may miss. Do not start the next phase until the PR is reviewed and merged.
